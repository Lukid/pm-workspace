## 1. Pre-Kickoff

### Documentazione
- [ ] Contratto/Ordine firmato ricevuto
- [ ] Offerta tecnica disponibile
- [ ] Budget e milestone di fatturazione chiari
- [ ] Contatti cliente acquisiti

### Setup Interno
- [ ] Progetto creato su OpenMemo
- [ ] Repository GitLab creato
- [ ] Cartella progetto creata in `projects/[nome]/`
- [ ] `.project-context.md` compilato
- [ ] Team assegnato e informato

### Preparazione Kickoff
- [ ] Agenda kickoff preparata
- [ ] Invito inviato a tutti i partecipanti
- [ ] Materiali da presentare pronti

---

## 2. Durante il Kickoff

### Presentazione progetto
- [ ] Obiettivi confermati
- [ ] Scope discusso e validato
- [ ] Timeline presentata e condivisa
- [ ] Ruoli e responsabilit√† chiariti

### Aspetti operativi
- [ ] Canali comunicazione concordati
- [ ] Frequenza SAL concordata
- [ ] Referenti cliente confermati
- [ ] Accessi/credenziali richiesti

### Rischi e assunzioni
- [ ] Dipendenze identificate
- [ ] Rischi principali discussi
- [ ] Assunzioni validate con cliente

---

## 3. Post-Kickoff (entro 48h)

### Documentazione
- [ ] Verbale kickoff inviato
- [ ] Brief progetto finalizzato
- [ ] RAID log inizializzato
- [ ] Actions log con prime azioni

### GitLab Setup
- [ ] Label create/verificate
- [ ] Board configurata
- [ ] Milestone create
- [ ] Prime issue inserite
- [ ] Team invitato con ruoli corretti

### Comunicazione
- [ ] Email riepilogo a team interno
- [ ] Condivisione materiali con cliente
- [ ] Prossimi passi confermati

---

## 4. Prima Settimana

- [ ] Pianificazione dettagliata prima fase
- [ ] Inizio analisi/design
- [ ] Primo check interno con team
- [ ] Verifica accessi e credenziali funzionanti
