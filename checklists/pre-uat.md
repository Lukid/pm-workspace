## 1. Sviluppo Completato

- [ ] Tutte le feature in scope sono in stato "Done"
- [ ] Nessuna issue bloccante aperta
- [ ] Code review completate
- [ ] Definition of Done rispettata per tutte le issue

---

## 2. Ambiente Staging

- [ ] Ambiente staging aggiornato all'ultima versione
- [ ] URL staging: [https://...]
- [ ] Dati di test presenti e realistici
- [ ] Configurazioni allineate alla produzione
- [ ] Performance accettabili

---

## 3. Test Interni Completati

- [ ] Test funzionali eseguiti dal team
- [ ] Test responsive (mobile/tablet/desktop)
- [ ] Test cross-browser (Chrome, Firefox, Safari, Edge)
- [ ] Bug critici risolti
- [ ] Buglist interna svuotata o con solo minor/cosmetic

---

## 4. Documentazione UAT Pronta

- [ ] UAT Plan preparato
- [ ] UAT Checklist preparata (personalizzata per progetto)
- [ ] Buglist template pronto
- [ ] Credenziali test preparate

---

## 5. Comunicazione al Cliente

- [ ] Email con istruzioni UAT inviata
- [ ] Credenziali di accesso comunicate
- [ ] Date e scadenze confermate
- [ ] Processo segnalazione bug spiegato
- [ ] Call di avvio UAT schedulata (se necessaria)

---

## 6. Team Pronto

- [ ] Dev disponibile per supporto e fix
- [ ] PM disponibile per triage e coordinamento
- [ ] SLA di risposta definiti e comunicati
